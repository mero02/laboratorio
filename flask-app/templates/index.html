Introducción

En los entornos actuales de redes y sistemas, el monitoreo cumple un rol fundamental para garantizar la disponibilidad, el rendimiento y la seguridad de la infraestructura. Históricamente, este monitoreo se ha basado en tecnologías como SNMP y Syslog, orientadas principalmente a dispositivos y servicios de red.

Sin embargo, con la adopción de arquitecturas distribuidas y microservicios, surge la necesidad de contar con mecanismos que permitan comprender el comportamiento interno de las aplicaciones, dando lugar al concepto de observabilidad.

Este proyecto implementa un laboratorio práctico que integra ambos enfoques, demostrando cómo el monitoreo clásico y la observabilidad moderna no se reemplazan, sino que se complementan.

Arquitectura General del Sistema

El laboratorio se ejecuta íntegramente sobre contenedores Docker conectados mediante una red bridge dedicada. A nivel conceptual, el sistema se divide en dos bloques arquitectónicos principales.

Arquitectura de Monitoreo Clásico

Este bloque representa el enfoque tradicional de administración y monitoreo de redes, centrado en la infraestructura y los servicios de sistema.

  • cliente1: Actúa como dispositivo monitoreado, ejecutando un agente SNMP y enviando logs mediante Syslog.
  • monitor: Funciona como servidor central de monitoreo, recibiendo logs y realizando consultas SNMP.

La comunicación se basa en protocolos ampliamente utilizados en entornos productivos: Syslog (UDP 514) para eventos y SNMP (UDP 161) para métricas.

Arquitectura de Observabilidad Moderna

Este bloque introduce un enfoque moderno orientado a la observación del comportamiento interno de las aplicaciones.

  • flask-app: Microservicio web desarrollado en Python con Flask, instrumentado con OpenTelemetry.
  • jaeger: Plataforma de trazas distribuidas utilizada para recolectar, almacenar y visualizar las trazas generadas.

Cada solicitud al microservicio genera trazas distribuidas que permiten analizar el recorrido completo de una petición, identificar latencias y detectar cuellos de botella.

Enfoques Complementarios

El laboratorio no presenta dos proyectos independientes, sino un único sistema integrado que combina ambos paradigmas.

  • El monitoreo clásico continúa siendo fundamental para la infraestructura, la red y los servicios de base.
  • La observabilidad se vuelve esencial para comprender el comportamiento de aplicaciones distribuidas y microservicios.
