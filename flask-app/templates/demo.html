Syslog Centralizado

Cliente1 envía logs vía UDP 514 al monitor. Los logs se almacenan en /var/log/cliente1.log.

Comandos para Prueba

Enviar log desde cliente1:

docker exec -it cliente1 bash

logger "Mensaje de prueba desde Cliente1"

Ver logs en monitor:

docker exec -it monitor bash

cat /var/log/cliente1.log

Evidencia

Capturas de pantalla de los comandos ejecutados y logs recibidos.

Monitoreo con SNMP

El contenedor monitor actúa como SNMP Manager y consulta métricas del contenedor cliente1, que funciona como SNMP Agent, utilizando SNMPv2c con la comunidad public.

Comandos para Prueba

Verificar agente SNMP activo en cliente1:

Confirma que el servicio snmpd se encuentra escuchando en el puerto UDP 161.

docker exec -it cliente1 bash

ss -lnup | grep 161

Consultar nombre del sistema (sysName):

Identifica el hostname del dispositivo monitoreado. Es útil para verificar identidad y alcance del agente.

docker exec -it monitor bash

snmpget -v2c -c public cliente1 1.3.6.1.2.1.1.5.0

Consultar tiempo de actividad del sistema (sysUpTime):

Muestra el tiempo desde que el sistema fue iniciado. Es una métrica básica de disponibilidad y detección de reinicios.

snmpget -v2c -c public cliente1 1.3.6.1.2.1.1.3.0

Consultar descripción del sistema (sysDescr):

Devuelve información general sobre el sistema operativo y la plataforma donde corre el agente SNMP.

snmpget -v2c -c public cliente1 1.3.6.1.2.1.1.1.0

Listar interfaces de red (ifDescr):

Permite visualizar las interfaces de red disponibles en el sistema. Es un ejemplo típico de monitoreo de infraestructura mediante SNMP.

snmpwalk -v2c -c public cliente1 1.3.6.1.2.1.2.2.1.2

Resolución de OIDs mediante MIB (snmptranslate):

Traduce un identificador simbólico definido en la MIB a su correspondiente OID numérico. Esto demuestra que el monitor cuenta con las MIBs correctamente instaladas.

snmptranslate -IR sysName.0

Consulta SNMP utilizando nombre simbólico (MIB):

Realiza una consulta SNMP usando el nombre definido en la MIB, en lugar del OID numérico. El resultado es equivalente, pero más legible para el administrador.

snmpget -v2c -c public cliente1 sysName.0

Recorrido de información del sistema (system group - MIB-II):

Muestra todas las variables del grupo system, que incluyen nombre, descripción, contacto, ubicación y tiempo de actividad del sistema.

snmpwalk -v2c -c public cliente1 system

Evidencia

Capturas de las consultas SNMP realizadas desde el monitor y las respuestas obtenidas (hostname, uptime, descripción del sistema e interfaces de red).
